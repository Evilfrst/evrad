SaaS — Comptabilité pour PME

Architecture applicative · API · Données · Sécurité
Python Django FastAPI PostgreSQL 2FA HTML/CSS
Application SaaS pour PME : facturation / gestion / compta simplifiée, avec sécurité renforcée (2FA) et base de données relationnelle.
Voir l’étude de cas
Objectif Proposer une solution web SaaS pour gérer les opérations comptables et administratives d’une PME (clients, factures, suivi).
Stack Python Django FastAPI PostgreSQL 2FA HTML/CSS Java
Architecture Application web structurée en couches :
  • UI : pages HTML/CSS (et composants Java si utilisés côté client)
  • Backend : Django (logique applicative) + FastAPI (endpoints API dédiés, intégrations ou services)
  • Données : PostgreSQL (modèle relationnel, contraintes, intégrité)
  • Sécurité : authentification avec 2FA (réduction du risque d’accès non autorisé)
Décisions clés
  • PostgreSQL pour la robustesse relationnelle (transactions, contraintes, cohérence des données)
  • Django pour accélérer la livraison (structure, admin, sécurité applicative)
  • FastAPI pour exposer une API moderne (services séparés / intégration future)
  • 2FA pour renforcer l’accès (compte utilisateur + second facteur)
Lien GitHub — saaS-comptable-pme

Évolutions “architecture SI” possibles : rôles/permissions fines, audit log, sauvegardes, observabilité (logs/metrics), déploiement CI/CD, durcissement OWASP (headers, rate limiting, gestion secrets).
